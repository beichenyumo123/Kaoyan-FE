## 三、各功能详细说明

### 3.1 图形验证码

**对应后端**：`GET /api/auth/captcha` + 登录接口新增 `captchaCode`/`captchaUuid` 字段

**实现位置**：`src/views/AuthPage.vue`

**交互流程**：

```
页面加载 / 登录失败 → GET /api/auth/captcha → 渲染验证码图片
用户点击图片 / 刷新按钮 → 重新请求验证码
用户输入验证码 + 账号 + 密码 → POST /api/auth/login
  ├─ 成功 → 存储 token → 跳转 /community
  └─ 失败 → 显示错误 + 自动刷新验证码
```

**UI 设计**：
- 验证码输入框（flex-1） + 图片（140×42px）+ 右侧内联刷新按钮
- 图片使用 `object-contain` 完整显示，点击可刷新
- 输入限制 4 字符，`autocomplete="off"`
- 登录失败后自动调用 `fetchCaptcha()` 刷新

**注册成功优化**：
- 注册成功后触发 Toast 提示「注册成功，请登录」
- 自动将注册邮箱填入登录表单的 account 字段
- 用户只需输入密码和验证码即可登录

### 3.2 接口限流（429 处理）

**对应后端**：`@RateLimit` 注解，超限返回 `{"code": 429, "message": "请求过于频繁，请稍后再试"}`

**实现位置**：
- `src/api/index.ts` — fetch 封装层拦截 429
- `src/components/AppToast.vue` — 全局通知组件

**架构**：

```
请求 → api/request() → fetch()
  ├─ 自动注入 Authorization: Bearer {token}
  ├─ 收到 429 → dispatchEvent('app-toast', { type: 'warning', ... })
  └─ 返回解析后的 JSON

AppToast.vue → 监听 'app-toast' 事件
  └─ 弹出悬浮通知（amber 色），4 秒自动消失
```

**Toast 样式**：

| 类型 | 配色 | 图标 |
|------|------|------|
| `warning`（429） | `bg-amber-50 border-amber-200 text-amber-800` | AlertTriangle |
| `error` | `bg-red-50 border-red-200 text-red-800` | XCircle |
| `info` | `bg-white border-zinc-200 text-zinc-800` | Info |

**设计约束**：存量页面（CommunityHome、AuthPage 等）未强制迁移到 `request()` 封装，保持最小改动原则。新功能统一使用 `request()`。

**`request()` 签名**：

```typescript
async function request<T>(url: string, options?: RequestInit): Promise<{ code: number, message: string, data: T }>
```

### 3.3 小组群聊

**对应后端**：6 个 REST 端点 + WebSocket 群聊协议

**实现位置**：`src/views/GroupList.vue`、`src/views/GroupChat.vue`

**导航入口**：CommunityHome 顶部 Header，铃铛图标左侧的 `Users` 图标按钮，跳转 `/groups`

#### 3.3.1 群组列表页（GroupList）

**路由**：`/groups`

**功能**：
- 我的群组列表：卡片式展示，含群首字母头像、群名、简介、人数、群主名
- 加入群组：输入群组 ID + 点击「加入」按钮 → `POST /api/chat/groups/{id}/join`
- 创建群组：弹窗表单，群名称（必填）+ 群简介（可选）→ `POST /api/chat/groups`
- 加载骨架屏 + 空状态引导

**API 调用**：

| 操作 | 方法 | 端点 |
|------|------|------|
| 获取群组列表 | GET | `/api/chat/groups` |
| 加入群组 | POST | `/api/chat/groups/{groupId}/join` |
| 创建群组 | POST | `/api/chat/groups` |

#### 3.3.2 群聊房间（GroupChat）

**路由**：`/chat/:groupId`

**核心架构**：

```
onMounted
  ├─ await fetchCurrentUser()       // 获取当前用户
  ├─ await fetchGroupDetail()       // 获取群名称
  ├─ await fetchHistory()           // GET /api/chat/groups/{id}/messages（分页、倒序翻转）
  └─ connectWebSocket()             // ws://host/ws/chat/{groupId}?satoken={token}

消息生命周期：
  发送 → ws.send({content}) ───→ 后端处理（敏感词过滤 + 异步入库）
       │                              │
       └── 乐观更新（isMine: true）     └── 广播给同群其他在线用户
                                       └── 他人收到 → addChatMessage()（isMine 即时判定）
```

**消息展示**：
- 自己的消息：右侧对齐，`bg-zinc-900 text-white rounded-br-sm`
- 他人的消息：左侧对齐，`bg-zinc-100 text-zinc-900 rounded-bl-sm`，带用户名
- 系统消息：居中，`text-xs text-zinc-400 bg-zinc-100/80 rounded-full`（加入/离开通知 + 在线人数）
- 消息时间：`MM-DD HH:mm` 格式

**输入区**：
- 自适应高度的 textarea（max 128px）
- Enter 发送，Shift+Enter 换行
- 聚焦时输入框整体上移 2px + 边框高亮
- WebSocket 未连接时禁用输入

**关键设计**：`isMine` 标记在消息入列时计算，而非在模板中调用函数比较。避免了 Vue 响应式依赖追踪的边缘问题。

```javascript
// 入列时即判定
const checkIsMine = (userId) => {
  const myId = currentUser.value?.id
  if (myId == null || userId == null) return false
  return String(myId) === String(userId)
}
```

**退出群组**：Header 右侧「退出群组」按钮，群主退出会收到「群主不能退群」提示。

### 3.5 热门推荐（HN 热度算法）

**对应后端**：`GET /api/posts/hot?pageNum=1&pageSize=10`

**实现位置**：`src/views/CommunityHome.vue`

**算法**：`Score = (likeCount − 1) / (T + 2)^1.8`，后端定时任务每 10 分钟计算并写入 Redis ZSet

**前端改动**：
- 左侧边栏「板块导航」顶部新增「热门推荐」（`Flame` 图标）
- 移动端标签栏新增「热门」tab
- 选中时调 `GET /api/posts/hot`，取消选中恢复原有时间排序

**分页格式差异处理**：

| 指标 | 常规 API | 热门 API |
|------|---------|---------|
| 数据源 | `/api/posts/page` | `/api/posts/hot` |
| 分页字段 | `hasNextPage` | `pages` |
| 排序依据 | 时间倒序 | HN 热度衰减公式 |

兼容逻辑：
```javascript
hasMore.value = showHotPosts.value
  ? result.data.pageNum < result.data.pages
  : result.data.hasNextPage
```

**交互**：热门模式下的点赞、收藏、评论数拉取与普通帖子完全一致，切换板块自动退出热门模式。

## 五、请求处理全链路

```
HTTP 请求（新功能使用 api/request()）
  │
  ├─ 自动注入 Authorization: Bearer {token}
  ├─ Content-Type: application/json
  │
  ▼
  ├─ 后端 CorsFilter → Sa-Token 鉴权 → @RateLimit 限流
  ├─ Jackson 反序列化（XSS 清洗 + Long→String）
  ├─ @Valid 参数校验 → SensitiveWordUtil 敏感词过滤
  │
  ▼
  ├─ 响应 → request() 解析 JSON
  ├─ status === 429 → window.dispatchEvent('app-toast', { warning })
  └─ 返回 { code, message, data }

WebSocket 请求（GroupChat）
  │
  ├─ ws://host/ws/chat/{groupId}?satoken={token}
  ├─ HandshakeInterceptor（Sa-Token + 群成员校验）
  │
  ▼
  ├─ 发送 → ws.send({content})
  ├─ 接收 → onmessage → type: message/system/error
  └─ 错误 → dispatchEvent('app-toast', { error })

AppToast.vue（全局监听）
  │
  └─ 监听 'app-toast' 事件 → 悬浮通知 → 4 秒自动消失
```

## 六、Bug 修复记录

| 问题 | 根因 | 修复 |
|------|------|------|

## 七、Vite 配置

```typescript
// vite.config.ts
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8081',
      changeOrigin: true,
    },
    '/ws': {
      target: 'ws://localhost:8081',
      ws: true,          // 启用 WebSocket 代理
      changeOrigin: true,
    },
  },
},
```

WebSocket 连接使用相对协议自动适配开发/生产环境：
```javascript
const protocol = window.location.protocol === 'https:' ? 'wss:' : 'ws:'
const wsUrl = `${protocol}//${window.location.host}/ws/chat/${groupId}?satoken=${token}`
```
